# LeZao 平台 1.0 版本开发文档

## 项目概览
LeZao 是一个面向教育场景的 Scratch 作品管理平台，支持老师、学生和管理员三个角色，提供作品的创建、管理、查看等功能。

## 版本信息
- **版本号**: 1.0.0
- **发布日期**: 2026-03-27
- **开发状态**: 已完成
- **测试状态**: 已测试通过

## 核心功能特性

### 1. 多角色支持
- **学生端**: 查看和管理个人 Scratch 作品
- **老师端**: 查看和管理自己及学生的 Scratch 作品
- **管理端**: 查看和管理所有用户的 Scratch 作品

### 2. 作品管理
- **作品创建**: 支持通过 Scratch 编辑器创建新作品
- **作品保存**: 自动在文件名中添加来源标记（1=老师端，2=学生端，3=管理端）
- **作品查看**: 支持在新窗口打开 Scratch 编辑器查看作品
- **作品删除**: 支持删除不需要的作品
- **作品搜索**: 支持按作品标题搜索

### 3. 身份识别系统
- **智能识别**: 根据文件名中的标记自动识别作品来源
- **视觉区分**: 老师作品显示不同的标识和边框
- **分组展示**: 老师端自动将作品分为"我的作品"和"学生作品"

### 4. 统一界面设计
- **现代化 UI**: 采用渐变配色和流畅动画
- **响应式布局**: 支持桌面端和移动端
- **一致体验**: 三个平台使用统一的设计语言
- **稳定排版**: 优化文字布局，防止标签页切换时排版错乱

## 技术栈

### 前端
- **框架**: React 18 + TypeScript
- **路由**: React Router v6
- **样式**: CSS3 (统一样式文件)
- **API 调用**: Axios

### 后端
- **框架**: Express.js + TypeScript
- **数据库**: MongoDB
- **文件存储**: 本地文件系统
- **认证**: JWT (JSON Web Token)

### 其他
- **Scratch 编辑器**: 集成官方 Scratch 3.0 编辑器
- **开发工具**: VS Code, Git

## 核心技术实现

### 1. 文件名标记系统
**实现位置**: `backend/src/services/fileService.ts`

- **文件名格式**: `timestamp_random_title_clientMark.sb3`
- **标记规则**: 1=老师端，2=学生端，3=管理端
- **解析逻辑**: 按下划线分割文件名，提取最后一部分作为标记

### 2. 作品识别逻辑
**实现位置**: `backend/src/controllers/scratchProjectController.ts` 和 `frontend/src/pages/teacher/ScratchProjectList.tsx`

- **后端识别**: 读取文件时解析文件名中的标记
- **前端识别**: 优先使用 clientMark 判断，无标记时使用 isOwn 判断

### 3. 统一界面实现
**实现位置**: `frontend/src/components/ScratchProjectGrid.css`

- **固定布局**: 卡片高度固定，元素尺寸明确
- **Flex 布局**: 确保统计信息始终在底部
- **响应式设计**: 适配不同屏幕尺寸

### 4. 权限控制
- **学生**: 只能查看和管理自己的作品
- **老师**: 可以查看和管理自己及学生的作品
- **管理员**: 可以查看和管理所有用户的作品

## 部署说明

### 前端部署
1. 构建前端项目：`npm run build`
2. 将构建产物部署到静态文件服务器
3. 配置域名和 SSL 证书

### 后端部署
1. 安装依赖：`npm install`
2. 构建后端项目：`npm run build`
3. 启动服务：`npm start`
4. 配置环境变量（数据库连接、JWT 密钥等）

### Scratch 编辑器集成
1. 将 Scratch 3.0 编辑器文件放置在 `scratch3-master` 目录
2. 配置编辑器的 API 调用地址
